# meshcore_region_info

Документация и руководства по системе регионов MeshCore.

## Кратко о регионах

Система регионов позволяет ограничить область видимости пакетов в mesh-сети. Каждый репитер объявляет свой регион через `region home` и настраивает правила пересылки через `allowf`/`denyf`.

**Зачем:** без регионов каждый пакет флудит всю сеть. С регионами городской чат остаётся в городе, краевой — в крае, а общероссийский доходит до всех.

**Как работает:**
- Отправитель указывает scope (регион) при отправке в канал
- Репитер проверяет свой allow/deny список
- Пакет без scope попадает под шаблон `*`
- `region default` подставляет scope в служебные пакеты репитера (DM, анонсы), но НЕ в транзитные

**Ключевые команды:** `region put`, `region home`, `region default`, `region allowf`/`region denyf`, `region save`

### Почему без `region home` и `region default` всё плохо

Если на репитере не указаны `region home` и `region default`:

1. **Нет маршрутизации** — без `region home` соседние репитеры не знают через какой узел reachable конкретный регион. Пакеты со scope будут пересылаться наугад или теряться, особенно на сетях с тремя и более репитерами.

2. **Служебка без scope** — без `region default` собственные DM, анонсы и запросы репитера уходят без scope. Если позже закрыть `*`, вся служебная связь репитера молча умрёт.

3. **Нет приоритета пересылки** — репитер не знает какой регион "родной", поэтому не может отличить локальный трафик от транзитного. При загрузке канала нет гарантии что местные пакеты приоритетнее.

4. **Клиенты не видят друг друга** — без анонсов `region home` клиенты в разных частях сети могут не обнаружить друг друга, даже если каналы пересылки открыты.

**Итого:** `region home` = репитер объявляет где он находится. `region default` = репитер помечает свою служебку правильным scope. Обе команды обязательны, даже если не планируешь закрывать `*`.

---

### Почему НЕ надо блокировать `*` на начальном этапе

Шаблон `*` (по умолчанию `allowf`) обрабатывает пакеты без scope. Если сделать `denyf *` слишком рано:

1. **Транзитные пакеты умрут** — `region default` помечает только собственные пакеты репитера (DM, анонсы). Чужие пакеты, проходящие через репитер транзитом, остаются без scope и будут отброшены.
2. **Устройства без scope отпадут** — пока все отправители не научились указывать scope в каналах, их сообщения попадают под `*` и молча исчезнут.
3. **Диагностика кошмар** — пакеты теряются без ошибок, понять почему что-то не доходит будет крайне сложно.

**Правильный порядок:**
1. Настроить `region home` и `region default` на всех репитерах
2. Попросить пользователей указывать scope в каналах
3. Подождать пока основная масса трафика будет со scope
4. Только потом — `denyf *` и `region save`
