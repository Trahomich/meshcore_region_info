# 1. Руководство по регионам MeshCore

Регионы MeshCore введены для ограничения нагрузки на mesh-сеть. Благодаря регионам сеть содержит меньше лишних промежуточных узлов (hops). Начиная с v1.15.0 регионы применяются к каналам, объявлениям (DM) и другим исходящим пакетам через команду `region default`.

**ВАЖНО посмотреть сначала:** Наглядное видео о регионах: https://www.youtube.com/watch?v=2LPwjfuCr3Y

# 2. Основная концепция

1. Репитер добавляет регионы
2. Репитер добавляет правила разрешения флуда (allow flood)
3. Пользователь клиентского узла добавляет флаг региона к сообщению в канале
4. Выбранный регион применяется к отправленному сообщению
5. Только репитеры, содержащие указанный флаг региона, будут повторять это сообщение

# 3. Важные замечания

- Начиная с v1.15.0 регионы применяются не только к каналам, но и к исходящим пакетам репитера (DM, объявления) через `region default`.
- По умолчанию репитер имеет регион `*`. Этот `*` означает, что репитер будет пересылать сообщения без флага региона. Это предотвращает блокировку сообщений и позволяет публичным каналам иметь глобальную область действия.
- В v1.16.0 добавлена команда `region def` для создания иерархии регионов в одну строку вместо множества `region put`.
- В v1.16.0 добавлены переменные `flood.max.unscoped` (ограничение hops для пакетов без scope, по умолчанию 64 = выкл) и `flood.max.advert` (ограничение hops для advert-пакетов, по умолчанию 8).
- Существует иерархия регионов. `ru` (Россия) — это область, `ru-kda` (Краснодарский край) — подобласть, `ru-kda-krd` (Краснодар) — подобласть третьего уровня. Пакеты со scope `ru-kda` будут пересылаться репитерами с регионом `ru-kda` и всеми его родителями (`ru`, `*`).
- Команда `region put` начиная с v1.15.0 автоматически разрешает флуд (allowf) для создаваемого региона.

# 4. Установка региона на репитере

Есть два способа установить регион на репитере:
1. Командная строка
2. Приложение

## 4.1 Что это значит?

- **Region**: Регионы, применяемые к репитеру
- **Allowf**: Означает разрешение флуда/пересылки из указанного региона
- **Home**: Объявление местоположения репитера — влияет на маршрутизацию
- **Default**: Scope по умолчанию для исходящих пакетов репитера (v1.15.0+)

## 4.2 Установка региона через командную строку

Можно добавить более одного региона. Это рекомендуется, если вы находитесь в районе, граничащем с несколькими регионами.
Например: мой репитер расположен в Нидерландах — регион Лимбург. Поскольку Лимбург граничит с другими регионами, рекомендуется разрешить ближайшие регионы (Северный Брабант).

Установка региона:
```
region put nl-li
region put nl-nb
region save
```

Автоматический allowf включён с v1.15.0 — отдельная команда `region allowf` обычно не нужна.

**Быстрая установка через `region def` (v1.16.0+):**

Вместо нескольких `region put` можно создать всю иерархию одной командой:
```
region def ru ru-kda ru-kda-krd
region save
```

Для создания ветвящейся структуры:
```
region def ru ru-kda ru-kda-krd|ru-kda ru-kda-gk
region save
```

Это создаст: `ru` -> `ru-kda` -> `ru-kda-krd`, затем вернётся к `ru-kda` и создаст `ru-kda-gk`.

Подробнее о `region def` — в разделе [Команды управления регионами](#6-команды-управления-регионами).

# 6. Команды управления регионами

| Команда | Описание |
|---------|----------|
| `region` | Показать все регионы, флаги и иерархию |
| `region get <имя>` | Информация о конкретном регионе |
| `region put <имя> [родитель]` | Создать регион (auto-allowf с v1.15.0) |
| `region def <token> [<token> ...]` | Создать иерархию регионов одной строкой (v1.16.0+) |
| `region remove <имя>` | Удалить регион (сначала дочерние) |
| `region allowf <имя>` | Разрешить пересылку региона |
| `region denyf <имя>` | Запретить пересылку региона |
| `region home [имя]` | Объявить/показать местоположение репитера |
| `region default [{имя\|<null>}]` | Установить/сбросить scope для исходящих пакетов (v1.15.0+) |
| `region list <filter>` | Показать регионы (filter: allowed/denied, firmware 1.12+) |
| `region load` | Пакетная загрузка регионов (интерактивная, serial only) |
| `region save` | Сохранить изменения в энергонезависимую память |

## Команда `region def` (v1.16.0+)

Команда `region def` позволяет определить иерархию регионов в одну строку. Логический курсор начинает с `*` (корень).

**Токены:**

- `имя` — создать `имя` как дочерний регион текущего курсора (аналог `region put имя` с курсором как родителем). Курсор перемещается на `имя`.
- `имя|jump` (или `имя,jump`) — создать `имя` как дочерний курсора, затем переместить курсор на `jump` (должен уже существовать или быть создан ранее в этой команде). `jump` НЕ становится родителем `имя` — это способ вернуться вверх по дереву и начать новую ветку.

Каждый созданный регион по умолчанию имеет flood-allowed (как `region put`). Ответ — результирующее дерево (формат как у `region`). При ошибке ответ `Err - ...`, и регионы созданные до ошибки остаются.

**Пример — линейная цепочка:**
```
region def a b c d e
region save
```
Создаёт: `*` -> `a` -> `b` -> `c` -> `d` -> `e`

**Пример — ветвящееся дерево:**
```
region def a b c d|b e f
region save
```
Создаёт: `a` -> `b` -> `c` -> `d`, затем возвращается к `b` и создаёт `e` -> `f`.

Эквивалент:
```
region put a
region put b a
region put c b
region put d c
region put e b
region put f e
```

**Пример — плоский список (все дочерние `*`):**
```
region def a|* b|* c|* d|* e|* f
region save
```

**Пример — пограничный репитер (две ветки от общего корня):**

Пограничный репитер должен быть членом обоих регионов одновременно. Через `region def` это делается одним вызовом с ветвлением через `|`.

Репитер на границе Краснодарского края и Ставрополья (без вложенных городов):
```
region def ru ru-kda ru-kda|ru ru-sta ru-sta
region home ru-kda
region default ru-kda
region save
```

Разбор токенов:
1. `ru` — создаёт `ru` под `*`, курсор → `ru`
2. `ru-kda` — создаёт `ru-kda` под `ru`, курсор → `ru-kda`
3. `ru-kda|ru` — `ru-kda` уже создан (подтверждение листа), прыжок к `ru`
4. `ru-sta` — создаёт `ru-sta` под `ru`, курсор → `ru-sta`
5. `ru-sta` — подтверждение листа

Результат:
```
* F
ru F
  ru-kda F [HOME] [DEFAULT]
  ru-sta F
```

Репитер — член всех трёх регионов: `ru`, `ru-kda`, `ru-sta`. Пакеты со scope `ru-kda` доходят, со scope `ru-sta` — тоже. Advert-пакеты расходятся в обе стороны.

**С вложенными городами (более глубокая иерархия):**
```
region def ru ru-kda ru-kda-krd ru-kda-krd|ru ru-sta ru-sta-kmv ru-sta-kmv
region home ru-kda-krd
region default ru-kda
region save
```

Результат:
```
* F
ru F
  ru-kda F
    ru-kda-krd F [HOME] [DEFAULT]
  ru-sta F
    ru-sta-kmv F
```

Ключевой момент: общий родитель `ru` обязателен. Без него scoped-пакеты не пойдут из одной ветки в другую. Ветвление `|ru` возвращает курсор к `ru`, и обе ветки висят на одном корне.

**Плоская граница (оба региона от `ru`, без вложенных городов):**
```
region def ru ru-kda|ru ru-sta
region save
```

Репитер член трёх регионов: `ru`, `ru-kda`, `ru-sta`. Scoped-пакеты проходят между ветками через общего родителя `ru`.

**Важно для пограничных репитеров — `flood.max.*`:**

- `flood.max.advert` слишком низкий — соседние регионы не узнают о репитере. Для пограничных рекомендуется 10-12 вместо дефолтных 8, если hop-расстояние до соседних нод большое.
- `flood.max.unscoped` слишком низкий — транзитный трафик без scope обрежется на границе. Дефолт 64 обычно достаточен.
- Настраивается через `set flood.max.advert 12`.

**Ограничения:**
- Serial принимает до 160 символов на строку. Для больших деревьев — разбить на несколько `region def`.
- Курсор сбрасывается на `*` между вызовами `region def`, поэтому начинайте следующий вызов с `имя|предок` для позиционирования.
- `region def` НЕ очищает существующее дерево — если имя уже существует, его родитель обновляется. Для полной очистки используйте `region remove`.

## Переменные конфигурации флуда (v1.16.0+)

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `flood.max.unscoped` | Макс. hops для пакетов без scope. Пакеты превысившие лимит отбрасываются | 64 (выкл) |
| `flood.max.advert` | Макс. hops для advert-пакетов. Пакеты превысившие лимит отбрасываются | 8 |

Установка:
```
set flood.max.unscoped 5
set flood.max.advert 3
```
