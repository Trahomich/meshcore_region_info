# Область по умолчанию (Default Scope Region)

**Автор:** Scott Powell, Lead Firmware Developer  
**Дата:** 17 апреля 2026 г.  
**Оригинал:** [blog.meshcore.io](https://blog.meshcore.io/2026/04/17/default-scope)

---

ПРИМЕЧАНИЕ: Ознакомьтесь с предыдущей статьёй блога, если вы не знакомы с концепцией регионов/областей.

Начиная с прошивки v1.12.0, была доступна область видимости сообщений групповых каналов, но всё ещё остаётся ряд препятствий для более эффективного использования регионов, особенно для таких сетей, как в Германии, где пытаются отключить трафик без области видимости (т.е. `region denyf *`).

В следующем релизе (v1.15.0) появится новая концепция «области по умолчанию» (default scope) как для сопутствующих узлов (companion nodes), так и для повторителей/серверов комнат. Это позволит отправлять сообщения DM с областью видимости, запросы входа, запросы статистики и т.д. (даже в сети, где отключён неограниченный flood-трафик).

---

## Мобильное приложение

В приложении MeshCore версии 1.43.0 появится новый параметр на экране экспериментальных настроек. Здесь вы можете задать имя региона области по умолчанию.

После установки ВСЕ flood-пакеты, отправляемые сопутствующим узлом (например, объявления, DM/входы/запросы, когда путь неизвестен), будут привязаны к этому региону.

Область видимости группового канала, если она задана, имеет приоритет над областью по умолчанию. Таким образом, вы можете по-прежнему задавать (как правило) более узкий регион/область для отдельных каналов.

---

## Администрирование повторителя и сервера комнат

Начиная с v1.15.0 будет доступна новая CLI-команда:

```
region default {name | <null>}
```

Она устанавливает указанный регион в качестве области по умолчанию или очищает его, если указать `<null>`. Обратите внимание, что регион создаётся автоматически, если его ещё нет в списке регионов. Также автоматически выполняется неявное сохранение регионов (`region save`)!

Область по умолчанию применяется только к пакетам, исходящим от повторителя/сервера комнат. То есть только к объявлениям для повторителя. К объявлениям и сообщениям комнаты — для сервера комнат.

---

## Входы/Запросы/Команды и ответы

Начиная с v1.15.0 действует новое правило для ответов. Они исходят от повторителя/сервера комнат, но подчиняются следующему правилу:

- Если область видимости пакета запроса может быть определена (т.е. регион есть в его списке), то для ответа используется та же область видимости.
- В противном случае ответ отправляется без области видимости.

Это также означает, что на запрос без области видимости будет дан ответ без области видимости.

Пока что это более безопасный вариант, вместо того чтобы ответы по умолчанию использовали область по умолчанию. В этот переходный период существует слишком много способов, которыми администратор повторителя может «выстрелить себе в ногу» и заблокировать себе доступ к собственному повторителю!

---

## ДОПОЛНЕНИЕ: новый параметр по умолчанию для «put»

Команда CLI `region put ...` теперь по умолчанию устанавливает для нового региона статус «allow» для flood. Так что вам больше не нужно дополнительно выполнять `region allowf ...`.

## Автономные устройства

Для устройств с поддержкой Ripple GUI, таких как T-Deck, следующий релиз (v9.6) будет поддерживать область по умолчанию. На экране «Регионы» просто выберите обычный регион (или `*`) и на экране деталей выберите пункт меню:

(При необходимости создайте новый регион из основного списка регионов для области по умолчанию.)

Вернувшись на экран списка регионов, вы увидите символ `(S)` рядом с текущим регионом области по умолчанию.

## FAQ

**В: Как администратор повторителя, что мне следует установить в качестве области по умолчанию?**

Как правило, это должен быть крупный регион, например, ваш город. Или тот регион, в пределах которого вы хотите ограничить объявления (например).

**В: Как пользователь приложения/чата, что мне следует установить в качестве области по умолчанию?**

Это должен быть крупный регион, который безопасно охватывает как вас, так и контакта(ов), которым вы хотите отправить DM. И ваши сообщения, и возвращаемые ACK будут привязаны к этому региону.

**В: Я управляю региональными сборками прошивки MeshCore с оптимальными настройками по умолчанию для региона. Могу ли я настроить собственный регион по умолчанию в моих сборках?**

Да. В соответствующих файлах `.ini` просто добавьте в `build_flags` следующее: `-D DEFAULT_FLOOD_SCOPE_NAME='" .. "'` (с именем в кавычках).
