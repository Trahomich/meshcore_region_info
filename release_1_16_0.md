## Ограничение трафика без scope

Новая переменная `flood.max.unscoped` предназначена для ограничения не-скопированного (unscoped) flood-трафика. Как и `flood.max`, она отбрасывает flood-пакеты без scope, которые уже прошли заданное количество hops. По умолчанию значение 64 — фактически отключено.

```
set flood.max.unscoped 5
```

## Ограничение advert-трафика

Новая переменная `flood.max.advert` ограничивает hops для advert-пакетов. По умолчанию 8 — пакеты прошедшие 8+ hops отбрасываются.

```
set flood.max.advert 3
```

## Новый preamble

При использовании низких Spreading Factor радио имеет меньше времени для захвата входящего пакета. Это изменение даёт более длинный preamble для быстрых SF, снижая вероятность пропуска. Для SF <= 8 preamble теперь 32 символа.

## Улучшенное обнаружение регионов в мобильном приложении

Новая прошивка companion и последнее обновление приложения позволяют делать ad-hoc запросы к ближайшим репитерам без добавления их в контакты. Это упрощает обнаружение репитеров и регионов.

## Новая команда `region def`

Создание регионов через CLI было довольно многословным. В v1.16.0 добавлена команда `region def` с краткой нотацией для определения иерархии регионов, особенно с глубокой вложенностью.

Подробнее в [CLI Command Reference](https://docs.meshcore.io/cli_commands/#region-management-v110).

## Расширенная поддержка ACK

Новая прошивка поддерживает «extended ACK» — 6 байт вместо 4. Дополнительные байты содержат расширенный номер попытки и дополнительный случайный байт. Это подготовка к будущей функции приложения для более чем 4 попыток прямых сообщений. Пока что нужно обновить репитеры.

## Raw-пакеты companion

Новая прошивка companion позволяет приложениям формировать полный пакет: заголовки, путь и payload. Это работает в связке с существующим RxLog — приложение может отправлять и принимать raw-пакеты.
